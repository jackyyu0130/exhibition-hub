Exhibition Hub V4.5｜完整替換最短流程

1. 先到 GitHub 專案首頁，點 Code → Download ZIP，保存舊版備份。
2. 解壓縮 Exhibition-Hub-V4.5-full-replacement.zip。
3. 複製 ZIP 內全部內容到儲存庫根目錄（與 index.html 同一層），覆蓋同名檔案。
4. 保留你原有的 CNAME 與 assets/hero-video.mp4（替換包不會覆蓋這兩個檔案）。
5. 提交到 main 分支；若用 GitHub Desktop，點 Commit to main → Push origin。
6. GitHub 網頁點 Settings → Pages → Source: GitHub Actions。
7. 點 Actions → Update data and deploy site → Run workflow → Branch: main → Run workflow。
8. 等待綠色勾勾，再開啟網站並按 Ctrl+F5（Mac：Cmd+Shift+R）強制更新。

不需要手動修改任何 HTML、CSS 或 JavaScript。

所有點擊位置、替換路徑、合併衝突處理、核對清單與復原方式：
